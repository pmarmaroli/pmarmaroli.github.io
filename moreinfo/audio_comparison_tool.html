About the Audio Comparison Tool

This web application enables users to upload multiple audio files and switch between them for easy comparative listening. The tool was designed for users who need a simple way to compare multiple audio files, allowing easy playback and sharing of audio content via a shareable key system.

Overview

Audio Comparison Tool facilitates the seamless comparison of audio files through a simple interface and interactive controls.

User Interface and Experience

  • Interactive Audio Controls: Users can play, pause, and switch between different audio files using an intuitive interface with waveform visualizations.
  • Waveform Display: Each audio file is displayed with a visual waveform, giving users more control over playback and navigation through the tracks.
  • Project Creation: Users can create their own projects by uploading multiple audio files, which are organized under a unique project name for future reference and sharing.
  • Project Selection: Users can select projects using a public key, allowing others to remotely access and listen to the uploaded audio files.

Backend Functionality

  • Audio File Handling: Audio files are uploaded via the user interface, and the server processes these files to make them available for comparison. The application supports multiple file uploads and different audio formats, with all audio files stored in Azure Storage containers.
  • Public Key Access: Users can choose a public key, similar to a password, which they can share with others. This key is linked to their audio project, allowing other users to access and listen to the shared audio files remotely by entering the provided key.
  • API Integration: The backend handles API requests to retrieve audio files, project data, and related labels for audio tagging from Azure Storage containers.

Post-Processing and Error Handling

  • Audio Label Management: Users can upload label files created with Audacity to tag specific time segments in the audio files. These labels are processed and displayed on the waveform, providing additional context during playback.
  • Error Handling: The application includes robust error handling to manage invalid public keys, failed audio uploads, and unsupported file formats.

Technical Stack and Tools

  • Frontend Development: The frontend is built using HTML, CSS, and JavaScript, incorporating WaveSurfer.js for audio visualization and control.
  • Backend Development: The backend handles file uploads, project creation, and API requests to retrieve audio and label data. It is integrated with the public key system for user access control.
  • Data Management: The server manages audio files and project data, ensuring that files are accessible based on user input and public keys.
  • Deployment: The application is deployed on the Azure (cloud) platform to handle file storage and access for multiple users.
