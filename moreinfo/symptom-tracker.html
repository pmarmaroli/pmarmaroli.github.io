About Symptom Tracker

Symptom Tracker is a simple web-based application designed to help users track and log their symptoms over time. It allows users to visualize the frequency of symptoms using charts, and download symptom logs as CSV files for further analysis.

Overview

Symptom Tracker offers a lightweight solution for individuals looking to monitor their health symptoms. It helps users add, log, visualize, and export their symptom data.

User Interface and Experience

  • Interactive Elements: The interface allows users to add new symptoms, log occurrences, and view a history of logs.
  • Chart Visualizations: Users can select a symptom from a dropdown menu to view a graphical representation of its occurrences over time.
  • CSV Download: Users can export their symptom logs to a CSV file for offline use or sharing with healthcare providers.

Frontend Development

  • Technologies Used: HTML, CSS, and JavaScript for building a responsive and interactive web application.
  • Chart.js Integration: The app uses Chart.js to generate dynamic line charts that display the frequency of logged symptoms over time.
  • Moment.js Integration: Moment.js is used to handle date and time formatting within the application.

Backend and Data Management

  • LocalStorage: Symptom Tracker uses the browser's LocalStorage to persist user data, allowing users to retain logs and symptoms across sessions without needing a backend server.

Post-Processing and Error Handling

  • Data Cleanup: A system is in place to allow users to delete symptoms and their associated logs, ensuring that they can maintain relevant data without clutter.
  • Error Handling: Robust error handling is implemented for actions such as data entry, log deletion, and CSV export.

Technical Stack and Tools

  • Frontend: HTML, CSS, JavaScript, Chart.js, Moment.js
  • Data Persistence: LocalStorage for session persistence
  • Deployment: Hosted on GitHub Pages
